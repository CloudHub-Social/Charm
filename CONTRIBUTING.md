# Contributing to Charm

Thanks for your interest in Charm 2.0! This is a ground-up rewrite of the Charm
Matrix client (`matrix-rust-sdk` over typed Tauri IPC), currently under active
pre-release development.

## Before you start

Scope, architecture, and design decisions are tracked in a private planning
doc, not in this repo. For anything beyond a small fix, please open an issue
first to confirm direction before investing time in a PR.

## Branching and PRs

- Branch off the latest `main`.
- **All PRs must target `main`** (or a `release/*` branch for backports) —
  a CI check and a local git hook both enforce this.
- Keep PRs focused; unrelated cleanup makes review harder.
- Fill out the PR template, including the AI-disclosure section if applicable.

## Identity — keep it clean

Charm 2.0 publishes as plain **Charm**. Don't reintroduce a version suffix
into any user- or store-visible identifier (package name, Tauri
`productName`/`identifier`, deep-link scheme, Cargo crate name). No `charm2`,
`charm-2.0`, `Charm 2`, etc.

## Quality gate

Run these before opening a PR — they mirror CI:

```sh
pnpm lint             # oxlint
pnpm fmt:check        # oxfmt --check
pnpm typecheck        # tsc --noEmit
pnpm test:coverage    # vitest run --coverage
pnpm knip             # dead-code / unused-dependency check
pnpm build            # tsc && vite build
```

Coverage thresholds in `vitest.config.ts` are a ratchet: if you add tests that
raise coverage, raise the floor in the same PR — never lower it to make CI
pass.

The Rust side (`src-tauri/`) has its own gate: `cargo fmt --check`,
`cargo clippy -D warnings`, `cargo test`.

Component stories (`src/components/ui/*.stories.tsx`) are checked for
accessibility violations via `pnpm build-storybook && pnpm test-storybook:ci`.

## IPC types

Types shared between Rust and TypeScript are generated by `ts-rs` into
`src-tauri/src/bindings/` (as a side effect of `cargo test --lib`) and
re-exported through `src/lib/matrix.ts`. Don't hand-edit a binding file —
change the Rust struct and regenerate. CI fails if committed bindings drift
from the Rust source.
